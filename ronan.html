Lists

Humans are natural list makers, and HTML provides elements for making up three types of lists:

  1. Unordered lists. Collection of items that appear in no particular order
  2. Ordered lists. List in which the sequence of the items is important
  3. Description lists. List that consist of name and value pairs, including but not limited to terms and definition

Unordered Lists

Identify an unordered list , mark it up as a < ul> element. The opening tag goes before the first list items, and closing tag < /ul> goes after the last item. Then, each item in the list gets marked up as a list item (li) by enclosing it in opening and closing li tags, as shown in this example. Notice that there are no bullets in the source document. They are added automatically by the browser. For example:

  • Fruits
    • Mango
    • Apple
    • Pineapple
  • Vegetables
    • Cabbage
    • Lettuce
    • Carrot

Ordered Lists

Ordered lists are for items that occur in a particular order, such as step by step instructions or driving directions. They work just like the inordered list described earlier, except they are defined with the ol element (for ordered list, ofcourse). Instead of bullets, the browser automatically inserts number before ordered list items, so you dont need to number them in the source document. Description Lists

Description lists are used for any type of name / value pairs, such as terms and their definition, questions and answers, or other types of terms and their associated information. Their structure is a bit different from the other two lists that we just discussed. The whole description list is marked up as a dl element. The content of a dl is some number of dt elements indicating the names and dd elements for their respective values. I find it helpful to think of them as "terms" (to remember the "t" in dt) and "definition(for the "d" in dd), even though that is only one use of description list in HTML5. For example:
